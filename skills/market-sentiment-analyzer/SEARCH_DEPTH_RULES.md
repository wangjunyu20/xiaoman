# 舆情搜索深度与规则规范 V3.1

## 搜索深度分级标准

### Level 1: 基础扫描（Quick Scan）
**用时**: 1-2分钟  
**目的**: 快速了解概况，适合常规监控

**搜索范围**:
- 实时新闻: 最近24小时内
- 分析文章: 最近3天内
- 搜索条数: 每个维度5-10条

**适用场景**:
- 日常盘前/盘后快速扫描
- 无重大事件时的常规监控

---

### Level 2: 标准深度（Standard）
**用时**: 5-10分钟  
**目的**: 全面收集信息，识别主要驱动因素

**搜索范围**:
- 实时新闻: 最近72小时内
- 分析文章: 最近7天内
- 产业报告: 最近30天内
- 搜索条数: 每个维度10-20条
- **必须覆盖**: 行业新闻、券商研报、政策动态

**适用场景**:
- 板块异动时的分析
- 日常研究报告生成

---

### Level 3: 深度挖掘（Deep Dive）
**用时**: 15-30分钟  
**目的**: 全面挖掘因果链条，识别根本原因

**搜索范围**:
- 实时新闻: 最近7天内
- 分析文章: 最近30天内
- 深度研报: 最近90天内
- 国际新闻: 最近30天内（地缘政治/能源）
- 搜索条数: 每个维度20-30条
- **必须覆盖**: 
  - 行业新闻
  - 券商研报（至少5家不同券商）
  - 政策解读
  - 国际形势（地缘政治/能源/美元）
  - 产业链上下游动态

**适用场景**:
- 板块大幅异动（涨跌幅>5%）
- 需要深度归因分析时

---

### Level 4: 全景扫描（Panoramic）
**用时**: 30-60分钟  
**目的**: 360度无死角全面分析

**搜索范围**:
- 不限时间（关键历史事件追溯）
- 不限来源（国内外主流媒体、智库报告）
- 搜索条数: 每个维度50+条
- **必须覆盖**:
  - 所有Level 3内容
  - 历史相似情境回顾
  - 专家观点汇总
  - 产业链全环节分析
  - 竞品/替代品动态
  - 监管历史案例

**适用场景**:
- 重大市场转折点的深度分析
- 策略报告/白皮书撰写

---

## 搜索规则要求

### 规则1: 5维强制覆盖
任何搜索必须覆盖以下5个维度，**缺一不可**:

| 维度 | 搜索关键词模板 | 最少条数 | 重要性 |
|-----|--------------|---------|-------|
| D1-实时新闻 | `{行业} 最新消息 今日 实时` | 10条 | P0 |
| D2-分析解读 | `{行业} 分析 研报 投资` | 10条 | P0 |
| D3-政策动态 | `{行业} 政策 监管 补贴` | 5条 | P1 |
| D4-国际形势 | `地缘政治 能源 美元 美联储` | 5条 | P1 |
| D5-资本市场 | `{行业} 主力资金 机构 北向` | 5条 | P1 |

**违反惩罚**: 缺少任一维度，搜索任务标记为"不完整"

---

### 规则2: 多源交叉验证
**关键信息必须至少3个独立来源验证**:

- 重大政策: 需官方媒体 + 券商解读 + 市场反应
- 业绩变动: 需公司公告 + 研报分析 + 股价反应
- 国际事件: 需国际媒体 + 国内解读 + 市场影响

**验证标记**:
- ✅ 已验证（3+独立来源）
- ⚠️ 待验证（1-2来源）
- ❌ 无法验证

---

### 规则3: 时间敏感性分级

| 信息类型 | 有效时间窗口 | 超时处理 |
|---------|------------|---------|
| 突发新闻 | 4小时内 | 标记为历史信息 |
| 市场分析 | 24小时内 | 降低权重 |
| 政策解读 | 72小时内 | 保留但标注时间 |
| 研报观点 | 30天内 | 正常参考 |
| 历史案例 | 无限制 | 用于类比分析 |

---

### 规则4: 关键词组合规范

**基础组合公式**:
```
核心词 + 限定词 + 时间词 + 来源词
```

**示例**:
- 基础: `光模块`
- 组合1: `光模块 最新消息 今日 财联社`
- 组合2: `光模块 下跌原因 分析 2026`
- 组合3: `CPO 主力资金 净卖出 东方财富`

**禁止的简单搜索**:
- ❌ `光模块` (过于宽泛)
- ❌ `AI新闻` (缺乏限定)
- ❌ `今天股市` (不聚焦)

**强制使用的限定词**:
- 行情: `涨跌/主力资金/成交量`
- 分析: `原因/分析/研报`
- 政策: `政策/监管/补贴`
- 国际: `地缘政治/能源/美元`

---

### 规则5: 搜索结果质量检查

**必须检查项**:

1. **相关性检查**:
   - [ ] 搜索结果与主题相关度>70%
   - [ ] 排除明显无关内容
   - [ ] 排除广告/推广内容

2. **时效性检查**:
   - [ ] 突发新闻在4小时内
   - [ ] 分析文章在有效期内
   - [ ] 历史信息标注时间

3. **完整性检查**:
   - [ ] 5个维度全部覆盖
   - [ ] 每个维度达到最少条数
   - [ ] 关键信息有多源验证

4. **准确性检查**:
   - [ ] 数据来源可靠
   - [ ] 事实性信息可交叉验证
   - [ ] 观点性信息标注来源

---

### 规则6: 搜索轮次要求

**最少搜索轮次**: 3轮

**第一轮: 广度扫描**
- 使用宽泛关键词，获取全景
- 目标: 识别主要话题和趋势

**第二轮: 深度挖掘**
- 针对第一轮发现的关键话题深入搜索
- 目标: 获取详细信息和深度分析

**第三轮: 补充验证**
- 针对第二轮发现的关键信息进行验证搜索
- 目标: 多源交叉验证，填补信息空白

---

## 搜索执行标准流程

### 执行前准备
1. **确定搜索深度级别**（根据事件重要性）
2. **准备关键词组合表**（按5维度）
3. **设置搜索时间范围**
4. **初始化验证矩阵**

### 执行中监控
1. **每轮搜索后检查**: 是否达到最少条数要求
2. **每维度搜索后检查**: 是否覆盖该维度关键信息
3. **发现关键信息后**: 立即启动验证搜索

### 执行后复盘
1. **质量检查**: 按规则5逐项检查
2. **完整性确认**: 5维度是否全部覆盖
3. **验证状态更新**: 标记已验证/待验证信息
4. **缺失信息记录**: 记录未能获取的信息及原因

## 违规处理

| 违规类型 | 处理方式 | 记录位置 |
|---------|---------|---------|
| 缺少维度 | 标记为"不完整"，强制补全 | 搜索日志 |
| 未达到最少条数 | 标记为"浅层"，降低置信度 | 质量报告 |
| 无交叉验证 | 标记为"待验证"，风险提示 | 验证矩阵 |
| 关键词过于简单 | 返回重新搜索 | 执行日志 |
| 超时才完成 | 标记为"延迟"，记录原因 | 任务记录 |
