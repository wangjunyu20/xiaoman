# Skill质量检查清单

## 使用说明
每个Skill开发或更新后，按此清单逐项检查。

---

## 一、元数据完整性 ✅

### 必填字段
- [ ] `name` - Skill英文名称（小写，单词间用-连接）
- [ ] `description` - 一句话描述（3-5行，说明用途和触发条件）
- [ ] `AIGC` 签名（如适用）

### 可选但推荐
- [ ] `allowed-tools` - 声明可使用的工具
- [ ] `metadata` - 额外元数据（环境变量、依赖等）

**检查标准**：
- description 不超过5行
- 包含明确的触发条件（"当用户...时使用"）

---

## 二、文档结构完整性 ✅

### 必须包含的章节

```markdown
1. 快速开始（Quick Start）
   - 3分钟上手的命令
   - 复制即可执行的示例

2. 核心功能（Features）
   - 能做什么
   - 使用场景

3. 执行流程（Workflow）
   - 怎么做
   - 步骤说明

4. 配置说明（Configuration）
   - 需要什么
   - 环境变量
   - 配置文件

5. 故障排查（Troubleshooting）
   - 常见错误
   - 解决方案

6. 参考文档（References）
   - 深入了解
   - 相关链接
```

**检查标准**：
- 至少有以上6个章节
- 每个章节有实际内容（非空）

---

## 三、渐进式披露设计 ✅

### Level 1: 一句话描述
- [ ] SKILL.md 的 description 字段精炼
- [ ] 用户一眼就知道这个Skill是做什么的

### Level 2: 快速开始
- [ ] 提供复制即可执行的命令
- [ ] 不需要理解原理就能使用

### Level 3: 详细说明
- [ ] 完整参数说明
- [ ] 使用示例

### Level 4: 原理和扩展
- [ ] 工作原理
- [ ] 高级用法
- [ ] 自定义配置

**检查标准**：
- 符合"先执行、后理解"原则
- 文档层次清晰

---

## 四、执行效果验证 ✅

### 必须有
- [ ] 成功案例示例（预期输出）
- [ ] 常见错误示例（错误信息+解决方案）
- [ ] 自检命令（快速验证配置）

### 推荐有
- [ ] 自动化测试脚本
- [ ] 执行日志记录
- [ ] 质量评分机制

**检查标准**：
- 新用户能根据示例验证Skill是否正常工作
- 错误信息有明确解决方案

---

## 五、代码和脚本 ✅

### scripts/ 目录
- [ ] 所有脚本可执行（有执行权限）
- [ ] 脚本有明确的入口点
- [ ] 包含错误处理

### 代码质量
- [ ] 有注释说明
- [ ] 有日志输出
- [ ] 异常处理完善

**检查标准**：
```bash
# 测试脚本可执行性
ls -la scripts/*.{py,sh,js}

# 测试脚本运行
./scripts/test.sh
```

---

## 六、references/ 目录 ✅

- [ ] 有接口文档或参考资料
- [ ] 文档之间有关联说明
- [ ] 不重复其他文档的内容

---

## 七、实际执行检查 ✅

### 执行测试
- [ ] 在干净环境中测试Skill
- [ ] 验证所有示例命令可执行
- [ ] 验证输出符合预期

### 边界测试
- [ ] 缺少依赖时的错误提示
- [ ] 配置错误时的错误提示
- [ ] 网络错误时的处理方式

---

## 八、评分标准

| 检查项 | 权重 | 评分 |
|--------|------|------|
| 元数据完整性 | 15% | /100 |
| 文档结构完整性 | 20% | /100 |
| 渐进式披露设计 | 20% | /100 |
| 执行效果验证 | 20% | /100 |
| 代码和脚本 | 15% | /100 |
| references目录 | 10% | /100 |

**总分 = (各项得分 × 权重) 之和**

| 总分 | 等级 |
|------|------|
| 90-100 | ⭐⭐⭐⭐⭐ 优秀 |
| 80-89 | ⭐⭐⭐⭐ 良好 |
| 70-79 | ⭐⭐⭐ 合格 |
| 60-69 | ⭐⭐ 需改进 |
| <60 | ⭐ 不合格 |

---

## 九、检查工具

### 自动检查脚本
```bash
# 检查文档结构
check_skill_structure.sh /path/to/skill

# 检查元数据
python check_metadata.py /path/to/skill/SKILL.md

# 执行测试用例
./scripts/run_tests.sh
```

### 手动检查清单
打印此清单，逐项打勾确认。

---

## 十、最佳实践

1. **先写文档，后写代码** - 确保文档和代码同步
2. **保持简洁** - 文档不是越多越好，关键是清晰
3. **示例驱动** - 每个功能都要有示例
4. **持续改进** - 根据用户反馈更新文档
