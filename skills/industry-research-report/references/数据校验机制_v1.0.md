# ✅ 数据校验机制

> 版本：v1.0 | 创建日期：2026-03-07
> 目标：确保报告数据真实、准确、可验证

---

## 校验原则

1. **真实性优先**：宁可标注"数据待确认"，绝不编造
2. **可追溯**：每条数据必须标注来源
3. **可验算**：数字类数据必须可验算

---

## 校验清单（发布前必检）

### 1. 资讯校验

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 日期 | 必须是 `YYYY-MM-DD` 格式 | ☐ |
| 日期有效性 | 不能是未来日期 | ☐ |
| 来源 | 必须有明确媒体/机构名称 | ☐ |
| 内容 | 与标题匹配，无断章取义 | ☐ |
| 重复 | 同事件不重复出现 | ☐ |

### 2. 股票数据校验

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 代码格式 | A股：`000001` / 美股：`QS` | ☐ |
| 名称匹配 | 代码与名称必须匹配 | ☐ |
| 股价格式 | 小数点后2位，如 `$12.34` | ☐ |
| 涨跌幅验算 | `(现价-昨日收盘)/昨日收盘 = 涨跌幅` | ☐ |
| 正负号 | 涨为 `+`，跌为 `-` | ☐ |

### 3. 格式校验

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 模糊词汇 | 无"近日""最近""近期" | ☐ |
| 投资建议 | 有数据支撑 | ☐ |
| 缺省数据 | 标注"待确认"或"暂无" | ☐ |

## 校验工具命令

### 1. 股票代码校验

**常用股票代码对照**：

| 行业 | 代码示例 | 名称 |
|------|----------|------|
| 量子科技 | 688027 | 国盾量子 |
| 量子科技 | 688567 | 国耀光电 |
| 量子科技 | 300270 | 亚光科技 |
| 美股量子 | QS | QuantumScape |
| 美股量子 | SLDP | Solid Power |

**校验方法**：
- 搜索 `[代码] + 股票` 确认名称
- 在同花顺/东方财富输入代码确认

### 2. 股价验算

```
涨跌幅 = (今日收盘价 - 昨日收盘价) / 昨日收盘价 * 100%

示例：
昨日收盘: $10.00
今日收盘: $10.50
涨跌幅 = (10.50 - 10.00) / 10.00 * 100% = +5.00%
```

### 3. 资讯日期校验

```
当前日期: 2026-03-07
资讯日期: 2026-03-07 ✓
资讯日期: 2026-03-08 ✗ (未来日期)
资讯日期: 2026-03-06 ✓
```

## 错误处理

### 发现错误怎么办？

| 错误类型 | 处理方式 |
|----------|----------|
| 数据错误 | 立即修正并标注"已更正" |
| 无法确认 | 标注"待确认" |
| 来源冲突 | 保留权威来源，标注争议 |
| 数据缺失 | 标注"暂无数据，待补充" |

### 禁止行为

| ❌ 禁止 | ✅ 正确做法 |
|---------|-------------|
| 估算股价 | 标注"获取失败" |
| 模糊日期 | 写"时间待确认" |
| 编造来源 | 写"来源待确认" |
| 复制旧数据 | 更新为最新数据 |

## 自动校验检查点

在生成报告时，必须经过以下检查：

```
1. 每条资讯
   ☐ 是否有日期？
   ☐ 日期格式是否正确？
   ☐ 来源是否标注？

2. 每个股票数据
   ☐ 代码是否存在？
   ☐ 名称是否匹配？
   ☐ 股价是否为数字？
   ☐ 涨跌幅方向是否正确？

3. 全局
   ☐ 是否有"近日"等模糊词？
   ☐ 投资建议是否有数据支撑？
```

## 错误追溯

如发现报告数据错误：

1. 定位错误数据
2. 追溯来源
3. 修正错误
4. 记录错误原因
5. 更新校验清单
