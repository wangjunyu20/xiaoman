# 测试体系 v1.0

> 三层检验机制，确保报告质量：格式正确、逻辑一致、业务合理

## 三层检验架构

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 3: 业务检验 (test_scenarios.py)                      │
│  ├── 利空消息存在性检查                                      │
│  ├── 舆情与资讯一致性检查                                    │
│  └── 新闻时效性检查                                          │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 逻辑检验 (test_consistency.py)                    │
│  ├── 数据勾稽关系验证                                        │
│  ├── 数据守恒验证                                            │
│  └── 情感标注一致性                                          │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 格式检验 (verify_reports.py)                      │
│  ├── 文件存在性                                              │
│  ├── 链接有效性                                              │
│  └── 内容非空性                                              │
└─────────────────────────────────────────────────────────────┘
```

**执行顺序**：Layer 1 → Layer 2 → Layer 3，任一失败即停止部署

## Layer 1: 格式检验

### 检验项

| 检查项 | 方法 | 失败处理 |
|--------|------|----------|
| 首页文件存在 | `os.path.exists('index.html')` | 终止部署 |
| 详情页数量 | 检查`reports/`目录文件数=15 | 终止部署 |
| 内容非空 | 文件大小>500字节 | 终止部署 |
| 新闻链接有效 | 正则匹配`https?://` | 标记警告 |
| 无模糊词汇 | 正则匹配"近日|最近|近期" | 标记警告 |

### 代码示例

```python
def verify_file_exists(filepath):
    """检查文件是否存在"""
    if not os.path.exists(filepath):
        raise VerificationError(f"文件不存在: {filepath}")

def verify_content_not_empty(filepath, min_size=500):
    """检查内容非空"""
    size = os.path.getsize(filepath)
    if size < min_size:
        raise VerificationError(f"内容过少: {filepath} ({size} bytes)")

def verify_news_has_links(html_content):
    """检查新闻有有效链接"""
    # 提取所有新闻链接
    news_links = extract_news_links(html_content)
    for link in news_links:
        if not link.startswith(('http://', 'https://')):
            raise VerificationError(f"无效链接: {link}")
```

## Layer 2: 逻辑检验

### 2.1 数据勾稽验证

**验证公式**：
```
采集总量 = 清洗输入
清洗输入 - 过滤 = 清洗输出
清洗输出 = 标注输入
标注输出 = 首页展示 + 详情页展示 + 归档
```

**代码示例**：

```python
def test_data_conservation(industry, lineage_log):
    """验证数据守恒"""
    # 采集 → 清洗
    collection_total = sum(
        s["count"] for s in lineage_log["collection"].values()
    )
    assert collection_total == lineage_log["cleaning"]["input"], \
        f"{industry}: 采集≠清洗输入"
    
    # 清洗计算
    cleaning = lineage_log["cleaning"]
    expected = cleaning["input"] - cleaning["invalid_url_removed"] \
               - cleaning["duplicate_removed"]
    assert expected == cleaning["output"], \
        f"{industry}: 清洗计算错误"
    
    # 标注 → 展示
    labeling = lineage_log["labeling"]
    distribution = lineage_log["distribution"]
    total_used = distribution["homepage"] + distribution["detail_page"] \
                 + distribution["archived"]
    assert labeling["input"] == total_used, \
        f"{industry}: 标注≠展示总量"
```

### 2.2 情感标注一致性

**验证逻辑**：
- 如果板块跌超3%，资讯列表中必须有至少1条利空
- 如果舆情极度负面(>70%)，资讯中利空占比应>20%

```python
def test_sentiment_consistency(industry, news_list, market_data):
    """验证情感标注与市场表现一致"""
    # 场景1: 大跌必须有利空
    if market_data['change_pct'] < -3:
        negative_news = [n for n in news_list if n['impact'] == '利空']
        assert len(negative_news) >= 1, \
            f"{industry}跌超3%但无利空消息"
    
    # 场景2: 情感分布合理
    if len(news_list) > 0:
        negative_ratio = len([n for n in news_list if n['impact'] == '利空']) / len(news_list)
        # 如果全是利好，但市场大跌，警告
        if negative_ratio == 0 and market_data['change_pct'] < -2:
            raise Warning(f"{industry}: 市场下跌但无利空消息")
```

### 3.1 利空消息存在性检查

**测试场景**：
```python
def test_negative_news_exists_when_market_drops():
    """
    场景: 当行业板块下跌超过2%时
    期望: 资讯列表中必须包含至少1条利空消息
    原因: 市场下跌必有原因，资讯应真实反映
    """
    for industry in industries:
        market_data = get_market_data(industry)
        if market_data['change_pct'] < -2:
            news = get_news(industry)
            negative = [n for n in news if n['impact'] == '利空']
            assert len(negative) >= 1, \
                f"{industry}板块下跌{market_data['change_pct']}%但资讯无利空"
```

### 3.2 舆情与资讯一致性检查

**测试场景**：
```python
def test_sentiment_news_alignment():
    """
    场景: 当股吧舆情极度负面(>70%)时
    期望: 资讯列表中利空占比应>20%
    原因: 避免"舆情说负面，资讯全利好"的矛盾
    
    注意: 允许差异，但不允许完全相反
    """
    for industry in industries:
        sentiment = get_sentiment_from_guba(industry)
        news = get_official_news(industry)
        
        if sentiment['negative_ratio'] > 0.7:
            negative_ratio_in_news = len([n for n in news if n['impact'] == '利空']) / len(news)
            assert negative_ratio_in_news > 0.2, \
                f"{industry}: 股吧极度负面但资讯利空占比仅{negative_ratio_in_news:.1%}"
```

### 3.3 新闻时效性检查

**测试场景**：
```python
def test_news_freshness():
    """
    场景: 检查新闻日期
    期望: 新闻日期应在最近7天内
    原因: 过期新闻无参考价值
    """
    today = datetime.now()
    for industry in industries:
        news = get_news(industry)
        for item in news:
            news_date = parse_date(item['date'])
            days_old = (today - news_date).days
            assert days_old <= 7, \
                f"{industry}: 新闻'{item['title']}'已过期{days_old}天"
```

### 3.4 数据重复检查

**测试场景**：
```python
def test_no_duplicate_news():
    """
    场景: 检查同一新闻是否重复出现
    期望: 同一URL或高度相似标题只出现一次
    原因: 重复新闻影响阅读体验
    """
    for industry in industries:
        news = get_news(industry)
        urls = [n['url'] for n in news]
        titles = [n['title'] for n in news]
        
        assert len(urls) == len(set(urls)), \
            f"{industry}: 存在重复URL"
        
        # 检查相似标题（简化版）
        for i, t1 in enumerate(titles):
            for t2 in titles[i+1:]:
                similarity = calculate_similarity(t1, t2)
                assert similarity < 0.8, \
                    f"{industry}: 新闻标题高度相似:\n{t1}\n{t2}"
```

## 集成到部署流程

### deploy_full.sh 集成

```bash
#!/bin/bash

echo "🚀 开始部署流程..."

# 1. 生成报告
echo "📄 生成报告..."
python scripts/generate_reports.py

# 2. Layer 1: 格式检验
echo "🔍 Layer 1: 格式检验..."
python scripts/verify_reports.py
if [ $? -ne 0 ]; then
    echo "❌ 格式检验失败，终止部署"
    exit 1
fi

# 3. Layer 2: 逻辑检验
echo "🔍 Layer 2: 逻辑检验..."
python scripts/test_consistency.py
if [ $? -ne 0 ]; then
    echo "❌ 逻辑检验失败，终止部署"
    exit 1
fi

# 4. Layer 3: 业务检验
echo "🔍 Layer 3: 业务检验..."
python scripts/test_scenarios.py
if [ $? -ne 0 ]; then
    echo "❌ 业务检验失败，终止部署"
    exit 1
fi

# 5. 生成检验报告
echo "📊 生成检验报告..."
cat > verification_report.txt << EOF
================================
报告检验结果 - $(date)
================================
格式检验: ✅ 通过
逻辑检验: ✅ 通过  
业务检验: ✅ 通过

数据血缘: 详见 data_lineage_report.json
================================
EOF

# 6. 部署
echo "📤 推送到GitHub..."
git add .
git commit -m "更新行业研究报告 - $(date +%Y%m%d)"
git push origin main

echo "✅ 部署完成"
```
